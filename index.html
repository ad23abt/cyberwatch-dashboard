🔬 Behaviour-Based Detection — No Feed Required

This engine detects suspicious activity purely from patterns in the log data — no external threat feed needed. It flags IPs that connect too frequently, access sensitive paths, use unusual ports, generate excessive errors, or connect at suspicious hours. This means it can catch zero-day threats that are not yet in any known feed.

🧠 System Design — How It Works

📌 Project Summary

This dashboard implements a Cyber Threat Correlation and Prioritisation System with two independent detection engines. Engine 1 uses feed-based correlation (matching log IPs against known bad indicators). Engine 2 uses behaviour-based heuristic detection (finding suspicious patterns in log data without any external feed). Together they provide comprehensive coverage — including threats not yet in any known feed.

🌐
1. Data Collection
Threat indicators fetched from Abuse.ch, AlienVault OTX, ThreatFox. Custom indicators can also be added manually.
📋
2. Log Ingestion
Server access logs are parsed. Users can add their own log entries manually or let the system simulate them.
🔍
3. Correlation + Behaviour
Engine 1 matches log IPs against feed. Engine 2 analyses behaviour patterns — both run independently.
🎯
4. Prioritisation
All detections scored 0–100. HIGH (70+) needs immediate action, MEDIUM (40–69) monitored, LOW (0–39) logged.

🔬 Why Behaviour Detection Matters

Feed-based systems only catch known threats. If an attacker is using a brand new IP not yet in any feed, feed matching will miss it entirely. Behaviour-based detection addresses this gap — it flags suspicious patterns such as too many connections, sensitive path access, or off-hours activity regardless of whether the IP is known. This is how enterprise tools like Darktrace and CrowdStrike catch zero-day threats.

🔢 Risk Scoring Formula

Feed Match base score: Ransomware +50, Malware C2 +40, Phishing +30, Botnet +25, DDoS +20
Feed bonuses: Multi-feed match +20, Repeat in log +15, Unusual port +10, Domain indicator +5

Behaviour scores: High frequency +40, Brute force pattern +35, Sensitive path +30, Excessive errors +25, Unusual port +20, Off-hours +15, Upload attempt +10

Scores capped at 100. 70–100 = HIGH, 40–69 = MEDIUM, 0–39 = LOW.
