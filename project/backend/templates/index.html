ML CLASSIFIER — DIRECT PREDICTION

Feed raw feature values directly into the RandomForest model. This bypasses the rule engine — you are talking to the ML model only. Use this to understand what the model treats as most suspicious.

ABOUT THE MODEL

Algorithm: RandomForestClassifier (sklearn)
Training data: 200 labelled rows — mixed threat types, benign traffic
Features: 10 (connection count, brute force hits, sensitive paths, error count, unusual port, off hours, upload attempts, feed match, feed count, threat type score)
Output: THREAT / BENIGN + confidence percentage (0–100%)
Retrain: python3 ml/train_model.py
Integration: Combined with live Abuse.ch feed and rule engine for final verdict.

API REFERENCE
GET /api/health
Health check — returns status, feed count, ML state
POST /api/analyze
Full analysis: Abuse.ch feed + behaviour rules + ML classifier
Body: {"ip":"...", "method":"GET", "path":"/login", "status":"200", "port":80, "hour":12, "log":[...]}
POST /api/batch-analyze
Analyse a full log array in one call, returns ranked results
Body: {"log": [{ip, method, path, status, port, hour}, ...]}
GET /api/feed-status
Abuse.ch feed cache status — indicator count, last fetch, errors
POST /api/refresh-feed
Force-refresh the Abuse.ch feed (bypasses cache)
GET /api/indicators
List up to 200 loaded threat indicators with full metadata
POST /api/predict-only
ML classifier direct — no rule engine, raw feature input
Body: {connection_count, brute_force_hits, sensitive_path_hits, error_count, unusual_port, off_hours, upload_attempt, in_feed, feed_count, threat_type_score}
